## Introducción
En esta parte implementaremos el sistema de piezas personalizadas, las categorías técnicas y la interfaz del configurador de monoplazas.

## Tipos de piezas y catálogo
El catálogo se divide en 4 grandes categorías, cada una con atributos técnicos exclusivos que se muestran en el frontend:
- Motor: potenciaCv, combustible, cilindrada y fabricante.
- Pintura: color hexadecimal, acabado y diseño.
- Suspensión: tipo (pushrod, pullrod), material, dureza y eje.
- Aerodinámica: Define el coeficienteArrastre, material y la zona(delantera, trasera, inferior).

## Cómo se asignan piezas a un coche
La asignación de piezas a un coche se gestiona a través de la entidad CochePieza y se controla desde el CochePiezaService.
El flujo en la base de datos es el siguiente:
1. El frontend envía el ID de la pieza que el usuario quiere equipar.
2. El backend comprueba si esa pieza está disponible en el catálogo.
3. Se verifica si el coche ya tiene una pieza instalada de la misma categoría. Si ya existe (por ejemplo, el coche ya tiene un motor), la fila en CochePieza se actualiza para apuntar a la nueva pieza seleccionada, reemplazando la que había antes. Si no existe, se crea una nueva asociación.

## Cómo se recalcula el precio
El cálculo dinámico del precio se realiza de manera centralizada en el backend cada vez que se añade, modifica o elimina una pieza.
- Precio Base: Cada coche tiene un precio base.
- Subtotal de Piezas: Es la suma del precio individual de cada pieza multiplicado por su cantidad (aunque en el configurador la cantidad es 1).
- Precio Total: Se actualiza sumando el Precio Base + Subtotal de Piezas, y se guarda directamente en la entidad Coche.

## Endpoints implementados
`GET /api/piezas/catalogo`: Obtiene todas las piezas marcadas como esCatalogo = true.
`GET /api/piezas/tipo/{tipo}`: Filtra las piezas disponibles por categoría (MOTOR, PINTURA, etc.).
`GET /api/coches/{cocheId}/piezas`: Devuelve la lista de piezas instaladas en un coche concreto.
`POST /api/coches/{cocheId}/piezas`: Asigna o reemplaza una pieza en el coche indicado y recalcula el precio.
`PUT /api/coches/{cocheId}/piezas/{id}`: Actualiza la cantidad o notas de una pieza ya instalada.
`DELETE /api/coches/{cocheId}/piezas/{id}`: Desinstala una pieza del coche y resta su importe del total.
