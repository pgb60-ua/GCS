# Memoria - Persona 3 (Javier)

## Tarea asignada
Detalle de coche, duplicar coche base y garaje del usuario.

## Flujo de duplicado de coche base
1. El usuario cliente inicia sesion.
2. Desde el catalogo de coches base, selecciona un coche y entra a la pantalla de detalle (`/coches/:id`).
3. Pulsa el boton "Personalizar".
4. El frontend llama a `POST /api/coches/{cocheBaseId}/duplicar` enviando el `usuarioId` del cliente logueado.
5. El backend crea un nuevo coche copiando los datos del coche base, pero con `esBase=false` y `precioTotal = precioBase`.
6. El frontend navega automaticamente a la pantalla de personalizacion (`/personalizar/:idNuevo`).
7. El usuario puede editar el nombre y descripcion del coche personalizado (PATCH).
8. En `Mi garaje` (`/garaje`), el usuario ve todos sus coches personalizados con precio total y fecha de creacion.

## Diferencia entre coche base y coche personalizado
- **Coche base**: `esBase=true`, no tiene `usuarioId`, no se puede modificar con PATCH. Son los modelos ofrecidos por el catalogo.
- **Coche personalizado**: `esBase=false`, tiene `usuarioId` asignado, se puede modificar con PATCH. Son copias de los coches base que el usuario puede editar y aĂ±adir piezas.

## Endpoints implementados por P3
- `GET /api/coches/base` - Listar coches base.
- `GET /api/coches/usuario/{usuarioId}` - Listar coches personalizados de un usuario.
- `GET /api/coches/{id}/resumen` - Resumen del coche con piezas y precio total calculado.
- `POST /api/coches/{cocheBaseId}/duplicar` - Duplicar un coche base para un usuario.
- `PATCH /api/coches/{id}` - Actualizar parcialmente un coche personalizado (nomenclatura, descripcion, imagenUrl).

- Faltaban archivos `.scss` para las nuevas paginas; se crearon vacios para que Angular compilara.
- El `home.page` de P1/P2 no mostraba coches base; se adapto temporalmente para que P3 pudiera probar el flujo completo.
