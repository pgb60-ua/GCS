# Memoria Tecnica - Persona 6: Ventas, checkout simulado e historial de compras

## Tarea asignada

Implementar el flujo de compra de coches personalizados: checkout simulado, registro de ventas en backend e historial de compras del usuario.

## Flujo de compra

1. El usuario inicia sesion.
2. Entra en un coche personalizado desde `/personalizar/:id`.
3. Pulsa `Comprar` y navega a `/checkout/:cocheId`.
4. El checkout carga `GET /api/coches/{id}/resumen`.
5. El usuario selecciona un metodo de pago demo.
6. El frontend crea la venta con `POST /api/ventas`.
7. Si la operacion es correcta, se redirige a `/compras`.
8. El historial muestra coche, fecha, metodo de pago, estado e importe.

## Checkout simulado

No se integra ninguna pasarela real. El metodo de pago se guarda como texto (`Tarjeta demo`, `Transferencia demo` o `PayPal demo`) y la venta se crea con estado `PAGADO` para facilitar la demo.

El importe se toma de `coche.precioTotal`. En la pantalla se muestra tambien el desglose del resumen del coche: precio base, piezas seleccionadas, cantidades, subtotales y total.

## Validaciones

Backend:

- Validar que exista el usuario.
- Validar que exista el coche.
- No permitir comprar coches base (`esBase=true`).
- No permitir comprar dos veces el mismo coche.
- Validar metodo de pago obligatorio.
- Permitir solo estados `PENDIENTE`, `PAGADO` y `CANCELADO`.

Frontend:

- Rutas protegidas con `AuthGuard`.
- Redireccion a `/login` si no hay usuario.
- Mensajes de error controlados si falla la carga o la compra.

## Endpoints de ventas

| Metodo | Ruta | Descripcion |
|--------|------|-------------|
| `GET` | `/api/ventas` | Lista todas las ventas |
| `GET` | `/api/ventas/{id}` | Obtiene una venta |
| `GET` | `/api/ventas/usuario/{usuarioId}` | Ventas de un usuario |
| `POST` | `/api/ventas` | Crea una venta |
| `PATCH` | `/api/ventas/{id}/estado` | Cambia el estado de pago |
| `DELETE` | `/api/ventas/{id}` | Elimina una venta |

DTO de entrada:

```json
{
  "usuarioId": "uuid",
  "cocheId": "uuid",
  "metodoPago": "Tarjeta demo"
}
```

DTO de salida:

```json
{
  "id": "uuid",
  "usuarioId": "uuid",
  "usuarioNombre": "Cliente Demo",
  "cocheId": "uuid",
  "cocheNombre": "RB20 personalizado",
  "montoTotal": 250000,
  "estadoPago": "PAGADO",
  "metodoPago": "Tarjeta demo",
  "fechaVenta": "2026-06-13T16:30:00.000+00:00"
}
```

## Pruebas manuales

Crear venta:

```bash
curl -X POST http://localhost:8080/api/ventas \
  -H "Content-Type: application/json" \
  -d '{"usuarioId":"ID_USUARIO","cocheId":"ID_COCHE","metodoPago":"Tarjeta demo"}'
```

Ver historial:

```bash
curl http://localhost:8080/api/ventas/usuario/ID_USUARIO
```

Cambiar estado:

```bash
curl -X PATCH http://localhost:8080/api/ventas/ID_VENTA/estado \
  -H "Content-Type: application/json" \
  -d '{"estadoPago":"CANCELADO"}'
```

Prueba final en frontend:

1. Crear coche personalizado.
2. Entrar en `/personalizar/:id`.
3. Comprar desde checkout.
4. Comprobar la venta en `/compras`.
5. Intentar comprar el mismo coche otra vez y verificar el error.
